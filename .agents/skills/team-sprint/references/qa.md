# QA (质量工程师) 工作指南

## 核心职责
通过系统化测试发现缺陷，给出质量评级，确保交付物达到验收标准。

## 工作清单

### 1. 测试策略设计
根据产物类型选择测试重点：
- **代码项目**：功能测试、边界测试、异常路径、回归测试
- **3D/视觉项目**：渲染正确性、动画流畅度、镜头完整性、资产加载
- **文档/内容项目**：准确性、完整性、一致性、可读性

### 2. 必做检查项

#### 通用检查
- [ ] 产物能否正常运行/加载？
- [ ] 是否有报错、警告、404？
- [ ] 输出文件是否生成在正确位置？
- [ ] 文件大小是否合理？

#### 功能检查
- [ ] 正向路径：正常输入是否得到预期输出？
- [ ] 边界路径：最小值、最大值、空值、超长值
- [ ] 异常路径：错误输入是否优雅处理？
- [ ] 并发/时序：如果有多个事件，顺序是否影响结果？

#### 视觉检查 (3D/前端)
- [ ] 所有镜头/页面是否完整，无元素丢失？
- [ ] 动画是否流畅，无跳变/卡顿？
- [ ] 不同角度/分辨率下是否正常？
- [ ] 资源加载失败时是否有降级？

### 3. Bug报告规范
每个bug必须包含：
```
ID: BUG-N
严重级别: P0/P1/P2/P3
标题: 一句话描述
复现步骤:
  1. ...
  2. ...
预期结果: ...
实际结果: ...
证据: 截图/日志/文件路径
```

## 输出模板

```markdown
## 测试报告
- 测试范围：...
- 测试环境：...
- 质量评级：A/B/C/D (A=无P0/P1, B=无P0, C=有P0未修复, D=无法运行)

## Bug清单
| ID | 级别 | 模块 | 摘要 | 状态 |
|----|------|------|------|------|
| BUG-1 | P1 | ... | ... | 待修复 |

## 回归测试记录
- 轮次1: X个bug → 修复Y个 → 剩余Z个
```
