# Tech Lead (技术经理) 工作指南

## 核心职责
确保技术方案合理、代码质量达标、技术债务可控。

## 工作清单

### 1. 架构评审
- [ ] 技术选型是否合理？（是否过度设计或设计不足）
- [ ] 模块划分是否清晰？耦合度如何？
- [ ] 扩展性：如果需求变化，哪些地方会爆炸？
- [ ] 依赖管理：外部依赖是否必要？版本是否锁定？

### 2. 代码审查 (Code Review)
必须逐文件检查：
- [ ] **正确性**：逻辑是否正确，边界条件是否处理
- [ ] **可读性**：命名是否清晰，注释是否必要且准确
- [ ] **一致性**：是否遵循项目编码规范（参考AGENTS.md）
- [ ] **性能**：是否有明显性能陷阱（O(n²)循环、内存泄漏等）
- [ ] **安全性**：是否有注入、越界、敏感信息硬编码
- [ ] **错误处理**：失败路径是否被处理，不会静默崩溃

### 3. 技术债务评估
- [ ] 是否有临时方案（TODO/hack）需要记录？
- [ ] 是否有重复代码需要抽象？
- [ ] 是否有可以简化的复杂逻辑？

## 输出模板

```markdown
## 技术方案评审
- 方案：接受/有条件接受/需重做
- 风险点：...

## 代码审查结果
| 文件 | 严重级别 | 问题 | 建议修改 |
|------|---------|------|---------|
| ... | P0/P1/P2 | ... | ... |

## 技术债务记录
1. [文件:行号] 问题描述 → 建议修复方式
```
